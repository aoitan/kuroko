# Review Fix Loop - Issue #22: ルールベースによるTODO/期限等の抽出

## Loop 1

### 1. 発見事項
- **TODO 抽出の精度**: `.*?(?:確認)する?` のようなパターンでは、「昨日確認した」などの過去形や「確認事項」などの名詞的使用を誤検知（偽陽性）してしまう。
- **MEETING と PENDING の重複**: 「調整中」が MEETING（「調整」を含むため）に誤検知される。
- **DEADLINE の網羅性**: 「今週中」「明日中」などが抽出されない。
- **相対日程「今週」の解決**: `今週: 0` となっており、当日扱いになっていた。

### 2. 重大度
- **中**: TODO の精度と MEETING の誤検知は、後続のフィルタリングや LLM 処理へのノイズとなる。
- **低**: 相対日程の解決精度。

### 3. 対応内容
- `kuroko/inference.py`:
    - TODO パターンを、文末（または句読点・空白の前）にある動詞や特定の接尾辞（`こと`, `ください`, `予定` など）に限定するように精緻化。
    - TODO 対象動詞に `提出`, `報告`, `共有`, `修正` などを追加。
    - MEETING パターンで「調整」の後に「中」が続く場合を否定先読みで除外。
    - DEADLINE パターンに `中`, `中に` などのバリエーションを追加。
    - `_resolve_relative_date` において「今週」を「その週の日曜日」として解決するように変更。
- `tests/test_inference.py`:
    - 偽陽性（過去形など）を検知しないことのテストを追加。
    - 「調整中」が MEETING にならないことのテストを追加。
    - 「今週中」の解決結果が正しいことのテストを追加。

### 4. 再確認結果
- `pytest tests/test_inference.py` を実行し、全 9 件のテストがパスすることを確認。

### 5. 残件
- より複雑な日本語表現（助詞が間に挟まる場合など）への対応は、必要に応じて後続フェーズで検討。

### 6. 収束判断
- 主要な誤検知パターンが修正され、テストで担保されたため、本フェーズ（Phase 5）としてのルールベース層の品質は十分と判断し、収束とする。
